`kor-legal-mcp` — 한국 법령·판례·행정해석 조회 **범용** MCP 서버 (구 `apt-legal-mcp`).

### 4. uv.lock 재생성 (`a384aba`)

### 5. FastMCP mount + lifespan + 에러 가시성 (`1422984`)
- **`/mcp` 307 Temporary Redirect**: `Mount("/mcp", inner)` 이중 prefix → `Mount("/", inner)`로 교체 (inner가 자체 `/mcp` 라우트 가짐)
- **`/mcp` 500 "Task group is not initialized"**: outer lifespan에서 inner lifespan을 `lifespan_context()`로 위임
- **빈 에러 메시지**: `LawApiError`에 예외 클래스명 + repr 포함, 3회 retry 추가
- **`ping()` flaky**: 단일 쿼리 → 여러 쿼리 시도, 하나라도 성공하면 OK

### 6. FastMCP DNS rebinding 비활성화 (`b5f4f00`)
- 포털 도메인이 `421 Misdirected Request`로 차단됨
- `TransportSecuritySettings(enable_dns_rebinding_protection=False)` — proxied 환경에선 무의미한 보호

### 7. Pydantic v2 validator 버그 수정 (`a8fe7b3`)
- `field_validator(...)(lambda _cls, v: ...)` — lambda는 classmethod가 아니라 인자가 한 칸 밀려 `v`에 `ValidationInfo` 객체 들어옴 → `.strip()` 폭발
- LLM이 처음 도구를 호출했을 때 드러난 런타임 버그 (import/스키마 구성 시점엔 안 터짐)
- 모든 6개 input 모델을 `Annotated[str, BeforeValidator(_non_empty)]` 패턴으로 교체
- Pydantic v2 idiomatic + boilerplate 제거

### 8. 판례 사건번호 lookup fallback (`5c17e5e`)
- LLM이 `search_precedent`로 `2021마6763` 받아 `get_precedent_detail`에 그대로 전달 → `PrecedentNotFound`
- 원인 1: `PrecedentSearchResultItem`이 `case_id`(판례일련번호) 미노출 — fast path 불가
- 원인 2: fallback search가 강제된 `search=2`(본문검색)로 사건번호 조회 — 본문에 사건번호 없음
- 수정: `LawApiClient.search_precedents`에 `body_search: bool = True` 파라미터, fallback에서 False, schema에 `case_id` 추가
- 새 진단 스크립트 `scripts/verify_tools.py`로 모든 6개 tool handler 회귀 검증

### 코드 / 빌드 함정 모음
- `field_validator(...)(lambda)` 사용 금지 — 반드시 `Annotated + BeforeValidator` 또는 `@field_validator + @classmethod`
- `FastMCP(version=...)` — 현 mcp SDK가 version kwarg 거부 → 제거됨
- `FastMCP`의 `streamable_http_app()`은 자체 `/mcp` 라우트를 가짐 → `Mount("/mcp", ...)` 금지, `Mount("/", ...)` 사용
- `streamable_http_app()`의 lifespan을 outer lifespan에서 반드시 위임 (`lifespan_context`)
- FastMCP `transport_security` 기본값이 loopback Host만 허용 → 프록시 뒤 배포는 `enable_dns_rebinding_protection=False` 필수
- `LawApiClient.search_precedents`: 일반 검색은 `body_search=True`(default), 사건번호 lookup은 `body_search=False`
